git 命令

# 从远程库克隆

git clone git@github.com:xxxxx.git

如果远程仓库没有添加 SSH,那么可以使用 https:// 的方式克隆,Git 支持多种协议,但 SSH 的速度最快.

# .git 文件过大,只clone最后一次的commit

在 clone 的命令后面加上 --depth=1
git clone ssh:xxxx --depth=1

# 全局写入

通常在第一次安装完 Git 后需要在全局中指定用户名和邮箱

git config --global user.name "yourName"
git config --global user.email "yourEmail@xx.com"

# 远程仓库,创建 SSH

创建 SSH Key.在用户主目录下,看看有没有.ssh 目录,如果有,再看看这个目录下有没有 id_rsa 和 id_rsa.pub 这两个文件,如果已经有了,可直接跳到下一步.如果没有,打开 Shell（Windows 下打开 Git Bash）,创建 SSH Key:

ssh-keygen -t rsa -C "youremail@xx.com"

你需要把邮件地址换成你自己的邮件地址,然后一路回车,使用默认值即可,由于这个 Key 也不是用于军事目的,所以也无需设置密码. 如果一切顺利的话,可以在用户主目录里找到.ssh目录,里面有id_rsa和id_rsa.pub两个文件,这两个就是SSH Key的秘钥对,id_rsa是私钥,不能泄露出去,id_rsa.pub是公钥,可以放心地告诉任何人.

# Git 仓库初始化

git init  # 如果不想一路回车可以使用默认 git init -y

# 添加文件到 Git 仓库

git add . (提交所有文件) git commit -m ‘message’ (message 是当前提交的说明<注释>)

git add .
git commit -m 'message'

# 查看仓库状态

git status

# 查看修改内容

git diff 就时查看 difference

git diff

# 查看日志

git log 命令显示从最近到最远的提交日志,如果嫌输出信息太多,看得眼花缭乱的,可以试试加上 –pretty=oneline 参数:

git log --pretty=oneline

# 版本回退

Git 必须知道当前版本是哪个版本,在 Git 中,用 HEAD 表示当前版本,上一个版本就是 HEAD^,上上一个版本就是 HEAD^^,当然往上 100 个版本写 100 个^比较容易数不过来,所以写成 HEAD~100.把版本回退到上一个版本,可以使用 git reset :

git reset --hard HEAD^ // 回退到上个版本

// 回到指定版本,0904ab是指版本号
// 版本回退后再找回回退前指定 commit id 的版本
git reset --hard 0904ab 
// 远程仓库的回退后需要强制提交回退远程版本
git push origin master --force

# 记录 commit id

git reflog

# 将本地仓库与远程仓库关联(如 github/gitee)

::: warning 注意: origin 后的内容为远程仓库地址 git remote add origin git@github.com:xxxxx.git :::

# 推送到远程库

git push -u origin master

如果远程仓库是空的,第一推送的时候 -u 参数会把本地的 master 分支内容推送到远程 master 还会把二者关联起来,以后就可以使用 git push origin master 简化命令.

# 创建/合并/删除分支

创建 dev 分支,然后切换到 dev 分支 git checkout 命令加上 -b 参数表示创建并切换,相当于 git branch dev , git checkout dev (branch 可以查看仓库的分支)

git checkout -b dev

合并分支,先切换到指定分支,如 master ,然后在 master 分支中 合并 dev 分支,如下两条指令

git checkout master
git merge dev

删除分支,如果需要删除dev分支可以使用如下指令

git branch -d dev

# 多人协作

要查看远程库的信息,用 git remote 或者，用 git remote -v 显示更详细的信息

现在有一个新的小伙伴,一起开发那么要怎么做呢?

// 克隆项目
git clone git@github.com:xxx/xxx.git
// 创建 dev 分支
git checkout -b dev origin/dev
// 添加所有修改文件
git add .
// 提交到仓库,注释 message
git commit -m "message"
// 拉取远程 dev 分支
git pull origin dev
// 如果 pull 失败,原因是没有指定本地 dev 分支与远程 origin/dev 分支的链接,根据提示,设置 dev 和 origin/dev 的链接
git branch --set-upstream-to=origin/dev dev
// 上传到远程 dev 分支
git push origin dev

# 标签

可以对版本打上标签,注明版本号,
// 创建标签,版本号 1.0
git tag v1.0
// 显示标签
git show
// 删除标签
git tag -d v0.1
